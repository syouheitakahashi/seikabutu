WEBサーバ構築の内容

  • オーソドックスなマルチAZのALB・webサーバ・データベースを利用した3層アーキテクチャ構成
  • ACMで証明書を発行し、Route53・ALBに連携し、セキュアなhttps通信を実現
  • DLMでwebサーバにおけるEBSのスナップショットを1日置きに自動取得する設定を実施
  • CloudWatch EventとSNSを利用してスナップショット取得後、Gmailにて通知
  • webサーバがCPU使用率80％を超過した場合AutoScallingでスケーリングし自動的に負荷分散
  • データベースはRDSを利用しマルチAZでのマスタースレーブ構成及び、リードレプリカを使用した読み込みの負荷分散を実現
  • CloudfrontとS3を利用した静的ウェブホスティングの構築及びWEBコンテンツ配信処理を高速化を実現
  • S3のライフサイクルにて30日後S3 Glacierへ自動的にアーカイブ
  • CloudWatchにて以下監視項目を設定しSNSでアラートをGmailへ通知するアーキテクチャを構築
  1. CPU使用率
    80%,90%,99%の3段階で設定し、さらに80%を下回った場合は回復した旨を通知するよう設定
  2. webサーバのステータス
    webサーバのEC2インスタンスのステータスが「実行中」以外の場合アラートを通知するよう設定
